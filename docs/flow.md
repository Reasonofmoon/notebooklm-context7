# Recommended Workflow

## 목표

문서를 먼저 근거 컨텍스트로 고정하고, 그 범위 안에서만 코드 결정을 내리는 개발 플로우를 만든다.

## 절차

1. Context7 선정
- 공식 문서
- 내부 API 스펙
- 아키텍처 ADR
- 에러 핸들링 가이드
- 스타일/코딩 컨벤션
- 배포/운영 규정
- 테스트 전략 문서

2. 컨텍스트 주입
- `nlm notebook create`
- `nlm source add` x 7

3. 구현 질의 템플릿
- "업로드된 문서 범위 안에서만 답변"
- "가정은 가정이라고 명시"
- "최종 출력은 체크리스트 + 코드 스켈레톤"

4. 검증
- 문서 근거 라인 확인
- 테스트 케이스 생성
- PR 체크리스트와 대조

## 실패 시 복구

- 인증 실패: `nlm login`
- 소스 처리 지연: `--wait` 유지 후 재시도
- 문맥 과대: Context7를 기능 단위로 다시 쪼개기
